How to do RPC Docs with a fork#

Before you begin, ask Lana or Karin to grant you access to the rpcdocs organization on GitHub.

Working with a fork provides an extra layer of "insulation" between your work in progress and the main repo. You can create branches in the main repo if you really want to, but note that many extra branches in the main repo can create clutter, in the form of branches that go stale and need to be pruned.

Forking the repository#

The procedure here uses https as an example. You can use ssh as well.

  1. In your browser, navigate to the rpcdocs repository.

  2. Click on Fork. When prompted, select your user ID.

  3. When the forking is complete, open a Terminal window. Create an rpcdocs folder and switch to that folder.

    $ mkdir rpcdocs
$ cd rpcdocs

  1. Clone your fork.

    $ git clone https://github.com/USERNAME/rpcdocs.git

  2. Switch to the new cloned folder.

    $ cd rpcdocs

  3. Set your upstream to rpcdocs/rpcdocs.

    $ git remote add upstream https://github.com/rpcdocs/rpcdocs.git

  4. Re-point your origin back to your fork.

    $ git remote set-url origin https://github.com/USERNAME/rpcdocs.git

Repeat these steps whenever you need access to one of the rpcdocs repositories.

Note

You can also set remotes to point to other users' forks. This is useful for more easily reviewing pull requests or collaborating on a change. For instance, if you want to create a remote for user janedoe:

$ git remote add janedoe https://github.com/janedoe/rpcdocs.git

Making sure you have the latest upstream#

  1. Navigate to the repo you want to update.

    $ cd REPONAME

  2. Fetch and merge the upstream code.

    $ git checkout master
$ git fetch upstream
$ git merge upstream/master

  3. Push the updated upstream code to your GitHub fork.

    $ git push origin master

    Note

    If you're feeling clever, you can cut and paste all those commands into a single line:

    $ git checkout master; git fetch upstream; git merge upstream/master; git push origin master

Do not forget the git push origin master step. Otherwise your fork on GitHub will fall behind the upstream repo, even though your local repo is up to date.

Editing, branching, and pull requests#

Before you begin, make sure you've grabbed the latest upstream code.

  1. Create a branch off master in which to do your work. A good naming convention for a branch is your initials, something indicating what you're fixing, and the date. For example: kl-configure-0930

    $ git checkout -b BRANCHNAME

  2. After you've finished your work locally, commit it. If it resolves an issue, be sure to include the issue number in the correct format.

    $ git commit

    Meaningful commit message on first line

Further explanation if needed

Fixes rpcdocs/issues#NNN

  3. Push the change to your fork on GitHub.

    $ git push origin BRANCHNAME

  4. In your browser, go to your forked repository.

    $ open https://github.com/USERNAME/rpcdocs

  5. At the top of the screen you'll see a status bar showing the change you've just committed and a green button to compare and create a pull request. Verify that the base branch is correct, then follow the prompts to create the pull request.

  6. Ask someone else to review, test build, and merge your commit.

Creating a pull request against a version branch#

Sometimes you may need to make a change on a particular version branch that affects only that version. This is the procedure.

  1. Check your git remote setup.

    $ git remote -v

    You should have something like this:

    origin https://github.com/USERAME/rpcdocs.git (push)
origin https://github.com/USERNAME/rpcdocs.git (fetch)
upstream   https://github.com/rpcdocs/rpcdocs.git (fetch)
upstream   https://github.com/rpcdocs/rpcdocs.git (push)

  2. Run the following command to grab the latest version staging branch (substitute the appropriate number for vN) and update it on your rpcdocs fork:

    $ git checkout vN-stage; git fetch upstream; git merge upstream/vN-stage; git push origin vN-stage

  3. Create a working branch off the version staging branch:

    $ git checkout â€“b BRANCHNAME

  4. Make your changes and commit them.

  5. Push the changes to your fork:

    $ git push origin BRANCHNAME

  6. Go to the rpcdocs repo page on GitHub and create a pull request. On the Compare changes page, make sure you have the following options selected:

    * base fork: rpcdocs/rpcdocs
* base: vN-stage
* head fork: USERNAME/rpcdocs
* compare: BRANCHNAME

  7. Complete the pull request as normal.
